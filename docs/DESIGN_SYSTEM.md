# 월급마을 디자인 시스템

> 시즌1. 이번달 생존기 — 레트로 도트 RPG × 싸이월드 미니홈피

## 1. 디자인 철학

월급마을은 **금융앱·가계부·SaaS 대시보드가 아니다.**

- 딱딱한 흰색 카드형 대시보드 금지
- 게임처럼 보이지만 실제 MMORPG가 아닌 **감성형 소셜 웹앱**
- 월급날까지 버티는 사람들의 **익명 도트 생존방** 커뮤니티
- 귀엽고 웃픈 월급 생존 분위기 — 공감과 유머가 핵심

### 참고 무드

| 요소 | 설명 |
|------|------|
| 레트로 도트 RPG | 픽셀 테두리, 도트 캐릭터, HP/상태 UI |
| 싸이월드 미니홈피 | 내 방, 방명록, 발자국, BGM 느낌의 따뜻함 |
| 생존 게임 메타포 | 생존등급, D-day, 퀘스트, 위기 카테고리 |

## 4. 픽셀 UI 규칙

### 픽셀 테두리

모든 카드·패널·버튼은 **계단형 box-shadow** 픽셀 테두리를 사용한다.

```
외곽: brown-700 (#4A3728) 2px
하이라이트: cream-100 상단/좌측 1px (입체감)
```

### 모서리

- `rounded-none` — 픽셀 UI는 둥근 모서리 사용 금지
- 모든 UI 요소는 직각

### 그림자

- 부드러운 drop-shadow 금지
- `shadow-pixel` — 4px 오프셋 단색 그림자만 허용

### 이미지 렌더링

```css
image-rendering: pixelated;
image-rendering: crisp-edges;
```

## 5. 레이아웃

### 모바일 우선

- 최대 너비: `max-w-md` (448px) — 앱 셸 중앙 정렬
- 하단 고정 네비게이션: 56px + safe-area
- 상단 헤더: 48px

### 간격

| 토큰 | 값 |
|------|-----|
| `gap-2` | 8px — 컴팩트 요소 |
| `gap-3` | 12px — 카드 내부 |
| `gap-4` | 16px — 섹션 간 |
| `p-3` | 12px — 카드 패딩 |
| `p-4` | 16px — 패널 패딩 |

### 그리드

- 홈 타일: 2열 그리드
- 피드: 1열 리스트
- 방명록: 1열 카드 스택

## 6. 공개 vs 비공개 정보

### 공개 (커뮤니티에 노출)

- 월급날까지 D-day
- 생존등급 (S~D)
- 오늘 상태 (텍스트/이모지)
- 최대 위기 카테고리 (예: 배달, 커피, 택시)
- 오늘의 한마디
- 캐릭터 상태 (HP 메타포)

### 비공개 (절대 UI에 노출 금지)

- 정확한 월급, 잔액, 카드값
- 지출 상세, 상호명, 위치정보, 대출정보

> UI 구현 시 mock 데이터에도 금액 수치를 넣지 않는다.

## 7. 캐릭터 & 일러스트

### 도트 캐릭터 상태

| 상태 | 표현 |
|------|------|
| 여유 | 밝은 표정, HP 80%+ |
| 보통 | 무난한 표정, HP 50~80% |
| 빡빡 | 땀, HP 30~50% |
| 위기 | 눈물/번개, HP 30% 미만 |

### 캐릭터 아바타

- 32×32 또는 48×48 CSS 픽셀 아트
- `CharacterAvatar` 컴포넌트로 통일
- `image-rendering: pixelated` 필수

## 8. 애니메이션

| 이름 | 용도 | 규칙 |
|------|------|------|
| `pixel-bounce` | 버튼 호버, 퀘스트 완료 | 2~3프레임, 0.2s |
| `pixel-blink` | D-day 강조 | opacity 토글 |
| `float` | 말풍선 | 2px 상하, 2s loop |

- 과도한 애니메이션 금지
- `prefers-reduced-motion` 존중

## 9. 아이콘

- `lucide-react` 사용하되 **stroke-width: 2.5**, 크기 16~20px
- 아이콘 단독 사용보다 픽셀 프레임 안에 배치
- 이모지 적극 활용 (🍜 ☕ 🚕 💸 🏠)
